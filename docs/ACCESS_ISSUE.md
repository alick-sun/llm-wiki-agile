# 飞书文档访问权限解决方案

## 问题分析
Bot创建的文件夹可能在bot空间，不在用户个人空间。

## 解决方案

### 方案A: 手动创建（推荐）

1. **在飞书中手动创建文件夹**
   - 打开飞书云文档
   - 创建名为 "LLM-Knowledge-Base" 的文件夹
   - 记录文件夹token（从URL中获取）

2. **使用现有文件夹继续**
   ```bash
   # 更新配置使用你创建的文件夹
   # 手动创建7个子文件夹
   # 继续使用wiki系统
   ```

### 方案B: 使用个人云空间根目录

直接在你个人的云空间根目录操作：
- 跳过文件夹创建步骤
- 直接在你的空间中创建子结构

### 方案C: Bot授权访问

如果你是bot的管理员：
1. 进入飞书管理后台
2. 找到应用 cli_aa880d3053f91bc2
3. 授权你的账号访问bot创建的文件

## 推荐方案

**手动创建 + 脚本辅助**：
1. 你手动创建主文件夹
2. 我提供脚本帮你创建子结构
3. 后续操作完全自动化

## 如何获取文件夹Token

1. 在飞书中打开你创建的文件夹
2. 复制URL
3. Token是URL中最后一部分
   例如: https://navinfo.feishu.cn/drive/folder/`RbCbfMRhWl3WnEddlb6c1RW5n4c`
                                                       ^^^^^^^^^^^^^^^^^^^^^^^^^
                                                       这就是token
